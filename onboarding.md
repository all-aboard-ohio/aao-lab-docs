# Contributor Onboarding Checklist

Welcome to AAO Data Lab! Complete these steps to go from new member to active contributor. Most people finish this in under an hour.

---

## Step 1 — Join the Community

- [ ] [Join the AAO Slack workspace](https://join.slack.com/t/all-aboard-ohio/shared_invite/zt-3wgj180pu-eWAJoGn4_6~y9YHR9Lq3qA)
- [ ] Introduce yourself in `#welcome` — your name, background, and what you hope to contribute
- [ ] Join `#dev-general`, `#dev-resources`, and `#announcements`
- [ ] Star the [All Aboard Ohio GitHub organization](https://github.com/all-aboard-ohio)

---

## Step 2 — Read the Essentials

- [ ] Read the [Contributing Guidelines](./contributing.md)
- [ ] Read the [Code of Conduct](./code-of-conduct.md)
- [ ] Skim the [Tool Stack](./tool-stack.md) to understand what we build with
- [ ] Skim the [Style Guide](./style-guide.md) for code and design conventions
- [ ] Review [Volunteer Roles](./roles.md) to find your best fit

---

## Step 3 — Set Up Your Environment

- [ ] Install Git, Node.js 18+, and VS Code (see [Getting Started](./getting-started.md))
- [ ] Fork and clone a repository you want to contribute to
- [ ] Install dependencies and run the project locally
- [ ] Confirm the local server starts without errors
- [ ] Ask in `#dev-resources` for any API keys you need

---

## Step 4 — Make Your First Contribution

- [ ] Browse issues labeled `good first issue`
- [ ] Comment on one to claim it
- [ ] Create a feature branch and make your change
- [ ] Open a Pull Request with the template filled in completely
- [ ] Respond to any review feedback promptly

---

## You Are In

Once your first PR is merged, you are an official AAO Data Lab contributor. As an active contributor, you have:

- A public contribution on your GitHub profile
- Your name in the project contributors list
- Access to more complex and impactful issues
- Eligibility for a letter of recommendation after 3 or more merged PRs (upon request)

Questions at any point? Ask in `#dev-general` on Slack. Everyone here was a first-timer once.
