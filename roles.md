# Volunteer Roles

AAO Data Lab is entirely volunteer-run. We organize contributors into four broad roles. You can contribute across multiple roles, and your role can evolve over time as you get more comfortable with the codebase.

---

## Software Developer

**What you build:** Web applications, data APIs, interactive maps, and visualizations.

**Skills:** JavaScript, TypeScript, React, Python, SQL, Node.js, REST APIs

**Good first issues:**
- Frontend component improvements labeled `good first issue`
- Bug fixes in existing tools
- API endpoint additions or fixes

**Progression:** Contributor → Reviewer → Maintainer

---

## Data Analyst / Data Engineer

**What you build:** Cleaned datasets, reproducible analysis notebooks, data pipelines, and automated reports.

**Skills:** Python (pandas, geopandas, numpy), SQL, Jupyter, dbt, data visualization (matplotlib, Plotly)

**Good first issues:**
- Issues labeled `data`
- Documenting existing datasets and their sources
- Data quality audits and anomaly reports

**Progression:** Contributor → Dataset Owner

---

## Designer (UI/UX)

**What you build:** Component designs, user flows, accessibility audits, visual assets, and the AAO Data Lab design system.

**Skills:** Figma, accessibility (WCAG 2.1), design systems, user research, mobile-first design

**Good first issues:**
- Issues labeled `design`
- Accessibility improvements (color contrast, keyboard navigation)
- Mobile responsiveness improvements

**Progression:** Contributor → Design Lead

---

## Project Manager / Researcher

**What you build:** Issue scoping documents, user stories, stakeholder summaries, policy landscape analyses, and project coordination artifacts.

**Skills:** Writing, research, project organization, transit policy knowledge, stakeholder communication

**Good first issues:**
- Issues labeled `research` or `documentation`
- Writing user stories for planned features
- Policy and dataset landscape summaries

**Progression:** Contributor → Project Coordinator

---

## Maintainer

Maintainers are experienced contributors who have demonstrated consistent, high-quality work. They:

- Triage and label new issues
- Review and merge pull requests
- Guide new contributors through onboarding
- Represent the project in external conversations

Maintainer status is invitation-based, offered after a sustained track record of quality contributions. If you are interested, reach out in `#dev-general` on Slack.
